# Git Workflow

## Before Committing

- Run `npm run build` and verify zero TypeScript errors
- Do not commit without a successful build

## Gitignored (do not commit)

- `dist/` — compiled output, regenerated by `npm run build`
- `.specs/` — local spec workflow state
- `node_modules/`
- `.zed/` — editor config

## Commit Conventions

- Use conventional commits: `feat(scope):`, `fix(scope):`, `chore(scope):`
- Scope should reflect the module or feature area (e.g., `spec-tools`, `tts`, `upstream`)
- Keep commits atomic — one logical change per commit

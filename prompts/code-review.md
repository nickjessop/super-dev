# Code Review

Before starting, ask me: **"What would you like me to review?"**

Wait for my response. I might point you at specific files, a git diff, unstaged changes, or a particular area of concern.

Once I've told you what to review, act as an outside senior engineer and perform the following:

1. Review the specified code critically — look for potential bugs, performance issues, security concerns, and adherence to best practices
2. Search the web for similar implementations to compare approaches and identify any improvements
3. Search the web for official documentation that validates the implementation patterns used
4. Provide a summary of findings including:
   - What was done well
   - Any concerns or potential issues
   - Recommendations for improvement based on industry standards and documentation
